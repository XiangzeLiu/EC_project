SSE Event Push System
基于 asyncio.Queue + StreamingResponse 实现：

· 每个 pending 节点维护一个专属 SSE Queue
· 管理员调用 PUT /nodes/{id}/approve 时，向对应 Queue 投递 event
· SE 端的 GET /nodes/{id}/events 长连接阻塞读取该 Queue
· 支持一个管理员审批 → 多个监听者同时收到通知
· 自动超时断开（SSE 长连接保护）
⚙️ SM 核心设计原则
  • 唯一信任根 — 所有 Token 签发和验证都在 SM 完成，SE 和 Client 都不自证身份
  • 内存+DB 双持久化 — NodeState 内存加速查询，SQLite 保证重启可恢复
  • 被动推送优于轮询 — SSE 事件驱动审批通知，零浪费等待
  • 分层防御 — 心跳超时(90s) → TCP探活 → 最终标记离线，避免误判
  • 外部服务解耦 — TT/IB 通过独立 service 封装，可随时替换底层实现
